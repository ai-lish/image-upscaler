🖼 圖片點樣變大？

由零開始了解呢個工具背後嘅原理。我哋會由「圖片 = 像素」講到 5 種放大算法，每個概念都附即時互動 demo — 你可以試下㩒㩒掣、拉下 slider，親身感受下。

1圖片 = 馬賽克牆

你見到嘅任何數碼圖片，無論係相、icon、定 emoji，內部其實都係一個「方格陣」：一個格 × 一個格 × 一個格⋯⋯排到完。

每一個格仔叫做「像素」(pixel)。你望住一張 1000 像素闊嘅相，代表佢由 1000 個格仔橫向排成。

📌 類比：想像你用 1000 粒相同大細嘅方塊磁磚砌一幅馬賽克牆，每粒磁磚得一種純色 — 砌出嚟就係一張圖。

👉 移個滑鼠入右邊嘅格仔試下：

滑鼠移入任何一格

2像素 = 三個數字

每個像素入面，淨係得三個 0 到 255 嘅數字 — 紅 (R)、綠 (G)、藍 (B)。電腦就係用呢三個色嘅混合，去模擬出所有顏色。

例如：R=255, G=125, B=49 就係橙色。

📌 類比：紅綠藍光係三支水彩筆，每支筆可以用 0 (一滴都唔落) 到 255 (落滿) 嘅份量。三支筆一齊落水 = 一個顏色。

3放大嘅難題

如果一張相係 100 × 100 像素，你想變成 400 × 400，電腦要點做？

佢冇辦法自己「畫」出新嘅細節 — 因為原本嘅資訊就係 100 × 100 嗰 10,000 格。

所以佢要用原本嗰 10,000 格嘅顏色，數學推算出 160,000 格嘅顏色。呢個過程叫做「重採樣」(resampling)。

📌 類比：你有一幅 10×10 嘅小畫，要將佢印到一張 40×40 嘅大紙度。你要喺原本 100 格以外嘅 1,500 格空白位置「猜」應該填咩色 — 但你只有原本 100 格嘅資料。
左：原圖 8×6 像素 　·　 右：放大 4× 後，多咗 192 格要靠「推算」填色

45 種「填色方法」並排比較

推算顏色有好多方法，由簡單到複雜，效果同速度都唔同。下面 5 格都係用同一張 16×16 細 icon 放大 8× 嘅結果 — 唯一分別係「推算法」。

📌 類比：5 個畫師用同一幅小畫去「照比例放大」。有人逐格抄最近嘅顏色，有人睇晒周圍先平均，有人用數學公式計權重 — 結果自然唔同。

5Lanczos 嘅核心：sinc 函數

Lanczos 係呢個工具預設嘅算法 (模塊 4 中間嗰格)。佢用一個叫 sinc 嘅數學函數，去決定「新格應該畀幾多分影響力畀周圍嘅舊格」。

sinc 嘅形狀係一個「中間高、兩邊上下振動、慢慢變零」嘅波浪。距離越近，新格受佢影響越大；距離越遠，影響越細。

而參數 a 就係「影響範圍」：

  • a = 2 = 影響半徑 2 個格，睇 4×4 = 16 格
  • a = 3 = 影響半徑 3 個格，睇 6×6 = 36 格
📌 類比：a 越大 = 越「民主」，問多啲鄰居意見先決定新色。a 細 = 較「獨斷」，淨係聽附近幾格就決定。
a 越大，覆蓋越廣，計算越慢，但效果越準

6後處理 = 執生

放大完之後，成件事未完 — 我哋仲會做 3 種「後處理」修飾，每種都係一個簡單嘅數學 filter，套用喺每個像素上面。

每個 filter 都係「3×3 鄰域 + 某個加權方式」：

  • Denoise 去雜訊：3×3 全部加起平均，再用原圖同平均色按比例混合 — 抹走雜訊
  • Contrast 對比度：每個像素 = pixel × c + 128 × (1-c) + brightness × 2.55
  • Sharpen 銳化：3×3 kernel [0,-1,0,-1,5,-1,0,-1,0] 加權 — 突出邊緣
📌 類比：沖完相之後再去 Photoshop 執：推一推「減少雜訊」滑桿、推一推「對比」滑桿、再「銳化」少少。

7點解唔係 AI 放大？

近年有種「AI 放大」(例如 Real-ESRGAN、waifu2x)，用神經網絡去「生成」原本冇嘅細節 — 效果可以非常驚艷。但本工具用純數學 (Lanczos) 嘅設計，係有意識嘅 trade-off：

本工具 (Lanczos)AI 放大 (Real-ESRGAN)
速度⚡ 即時 (CPU 都得)🐢 慢 (通常要 GPU)
私隱🛡 完全本地，零上傳☁ 多數要 upload 到 server
成本💰 完全免費💸 好多要訂閱
細節生成❌ 唔會「無中生有」✅ 可以生成新紋理
可預測✅ 純數學，每次一樣⚠ 偶有「幻覺」細節
檔案大小📄 單一 HTML ~50KB📦 連 model ~100MB+

簡單講：如果你要快、要私隱、要可預測，用本工具。如果你要極致畫質、肯等肯畀錢，AI 放大更適合。
